# Mode: apply — Live Assistant

Use this mode when the user is filling out:

- job application forms
- consulting portals
- freelance marketplaces
- proposal forms

## Workflow

1. Detect the company or client and the role or project.
2. Look for an existing report.
3. Load the prior context.
4. Identify the visible questions.
5. Generate copy-ready answers.

## Tone Rules

For English application notes, cover notes, and message fields:

- avoid stiff openings like `I am applying` unless the form requires blunt wording
- avoid `my background fits well` or similar generic fit language
- use natural transitions and a conversational rhythm
- make the reason for reaching out specific to the company, role, or project
- write like an experienced developer, not like a pasted summary from a CV

## After Submission

If the user confirms they submitted:

1. update the tracker status to `Submitted`
2. save any final notes
3. suggest `outreach` if follow-up makes sense
